## 🚀 Deploy to GitHub Pages

### Prerequisites:

1. **Create a new GitHub repository** named `geosignalagent` OR clone existing one
2. Push this minimal repo contents to it

### Setup GitHub Secrets:

Go to repository → `Settings` → `Secrets and variables` → `Actions`

Add these secrets:

| Secret Name | Example Value | Required |
|------------|---------------|----------|
| `POLYMARKET_API_KEY` | `pk_live_...` | ✅ Yes |
| `ALCHEMY_POLYGON_API_KEY` | `sk_live_...` | ⚠️ Optional |
| `DEMO_MODE` | `true` (initial deployment) | ⚠️ Temporarily |

### Enable GitHub Pages:

Go to: `Settings → Pages`

- Source: Select `gh-pages` branch (or `main` if configuring custom domain)
- Build and deploy on commit: ✅ Enabled
- Click "Save"


## 🔐 Security Architecture

This repository uses **server-side data fetching** for maximum security:

```
GitHub Repo Secrets (Private, Encrypted)
    ↓
GitHub Actions Runner (Fetches data with API keys)
    ↓
Pre-fetched JSON files in data/
    ↓
GitHub Pages CDN (Public frontend - NO secrets exposed)
```

Your API keys are stored securely in GitHub repo secrets and NEVER appear in:
- The public HTML/CSS/JS code
- Browser network requests
- Downloadable repository files
- Public GitHub Actions logs

## 🔄 Automated Deployment

### Schedule:

Data fetches **daily at 11pm UTC** (7pm Madrid time) via cron schedule.

To change schedule, edit `.github/workflows/deploy.yml`:

```yaml
schedule:
  # Example: Change to every 6 hours
  - cron: '0 */6 * * *'
```

### Manual Trigger:

Click "Run workflow" in `Settings → Actions` to force immediate data refresh.

## 🆘 Troubleshooting

**Dashboard not loading?**
- Check GitHub Actions logs in Settings → Actions for errors
- Verify secrets are set correctly
- Wait up to 10 minutes for first build

**Data not showing?**
- Workflow may not have run yet - check Actions tab
- If no data files created, trigger manual workflow run
- Ensure API keys are valid and prefixed correctly (`pk_live_`, `sk_live_`)
